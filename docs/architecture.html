How They Connect

MockIdentityRegistry

Agent registers on-chain identity (ERC721 token). The token ID becomes the agentId used across all contracts.

➔

MockReputationRegistry

Validators give feedback using the agentId from the identity registry. This is the "traditional" reputation layer.

➔

NeuralOracle

The oracle adds a BCI confidence layer on top. It scores HOW the validation was done, not just the result. Same agentId links all three.

Key Design Decisions

  • ● Ownable access control -- Only the oracle agent can submit scores, preventing manipulation
  • ● BCI data as hash -- Raw neural data stored off-chain, only the hash goes on-chain for verification
  • ● Gas-efficient aggregates -- totalScore and scoreCount tracked incrementally, no loops needed for averages
  • ● Score updates -- Resubmitting for the same agent recalculates aggregates correctly (subtract old, add new)
  • ● Solidity 0.8.27 -- Cancun EVM target with OpenZeppelin 5.x for modern security patterns
